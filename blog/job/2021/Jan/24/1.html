My Amazon Interview Experience

PT

Pritish Thakkar

Online Round

Amazon hosted the online round on Hackerearth platform. There were twenty MCQ’s based on computer science subjects like data structures, algorithms, Operating system, Database, Computer Networks, Theory of computation etc.

Two programming problems:

  1. Given an unsorted array. Find the minimum number of elements which can be removed from it to make it a sorted array.
  2. Given an array, find the maximum sum of non-consecutive elements. Link

F2F Round – 1

Only one question was asked during this round. Given two linked lists and those linked lists represent two numbers and you have to multiply them and then return a new linked which contains the product.

The problem was easy but I was nervous, so I made some mistakes. I thought that I will be rejected after this round but my code writing skills saved me. I practiced thousands of easy and medium problems on various online platforms and thus my hard-work saved me that day.

Takeaway: Practice writing clean code. Even if the problem you are solving is an easy problem just try to write it’s code. It will make you faster and help you achieve cleanliness in your code. Also, by practicing problems you will be learn to give sensible names to your variables.

F2F Round – 2

In this round three coding problems were asked from me.

  1. Problem Link
  2. Given a sorted matrix of 0’s and 1’s. By sorted I mean that matrix is sorted both row-wise and column-wise. Find the row with maximum number of 1’s. This was an easy version where the row with largest number of 1’s will be the last row.
  3. This is a harder version of second problem. In this the matrix is sorted row-wise but not column-wise. Try to solve this problem in O(n + m) time complexity, where n is number of rows and m is number of column in the matrix.

Again my clean code writing skills saved me. I had used reference variables(C++) in first problem, but my interviewer was not aware of reference variables. He knew about address operator and was thinking about reference variables the same way. So, I tried my best to convince him by giving various examples of passing a parameter to a function as reference using pointers and by reference variables.

Takeaway: You never know the expertise of your interviewer. For ex. in my case he may be expert in C but not in C++. So I would say that there is no need to learn all programming languages but instead try to master one.

F2F Round – 3

This interview round was my best round. As I entered the interview room, the interviewer started throwing operating system theoretical questions on me. Some of those questions were:

  1. What is the difference between a thread and a process?
  2. What is virtual memory?
  3. What is a race condition?
  4. What do you know about scheduling algorithms?

He also asked me to write code for two coding problems:

  1. Given a matrix. Shift the matrix k-times clockwise. Shifting means that you have to shift elements by k steps in clockwise direction.
  2. You are given an array A which contains ‘n‘ strings. Also a second array B will be given which will contain the lowercase characters which are important. Now you have to find a string from array A which will contain all the important characters and have the maximum count of important elements. If there are two or more answers for the problem, you have to print the string with lowest index. Desired complexity: O(n*max_length_of_strings_in_A)

Takeaway: Only programming skills are not important during an interview for software engineering position, but also the theoretical knowledge of computer science subjects is essential.

Telephonic Round – 4

Questions asked in this interview were mostly behavioral. The interviewer was from Kindle team. So, the first question he asked from me was “Have you ever used kindle?”.

Other questions asked from me were:

  1. Problem Link
  2. When do we use threads and when do we use processes.
  3. What are the problem with threads?
  4. What is the most challenging project you ever did?
  5. How you tested your project?
  6. What is the mistake you made recently?
  7. Why Amazon over your other internship companies?
  8. I told him that I had used AWS, so he asked me what have I used in AWS?
  9. Any other project that you did out of interest?
  10. What technologies you’d learnt during that project(out of interest).

Takeaway: Behavioral questions are sometimes the most difficult to answer. So, collect all the behavioral questions asked from people in the interviews and think the solution for them in advance.

Finally, I got the offer from Amazon.
